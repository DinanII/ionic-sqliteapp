This is an example of how SQLite can be used inside a mobile application (for IOS and Android) created using the Ionic framework. The SQLite community plugin is used to work with a local SQLite database.

Some important notices

- This app uses Angular 20, which is not the most recent version. Migrating to the most recent Angular version using the VSCode Webnative extention caused some dependency problems.

- The ionic cap add ios and ionic cap add android commands did not recognize the capacitor.config.ts configuration, so a JSON file is created instead.
The .ts file is keeped for reference purposes.

- The NPM package @capacitor-community/sqlite is used to work with SQLite.

- After initializing the project, I had to execute npm install @capacitor/ios and npm install @capacitor/android to create the appropiate IOS and Android projects.

- This project is based on https://devdactic.com/sqlite-ionic-app-with-capacitor. this tutorial is ouddated, so some things had to be done differently.

- To turn off the opt-out telemetry program: npx cap telemetry off

- This app does not have a web implementation, meaning a web browser can not be used to run the SQLite capabilities. You have to program your alternative yourself, with I.E. localstorage or the secure device string storage plugin.

- The SQLite plugin has to be registered into the MainActivity as well, this applies for the Android project and not IOS.
HTTPClient is used to download some mockup data for testing (registered in app.modules.ts). FALSE, or to some degree. This does not apply for newer versions from Ionic and the associated plugins, like the SQLite plugin we're using here.

- It is possible to manage multiple databases with CapacitorSQLite. A database name must therefore always be given when interacting with the database.

- A connection is required when interacting with the database, multiple connections to the same DB could cause problems when the db is locked by another connection while attempting to interact with it.
